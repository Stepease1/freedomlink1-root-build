# Lineage Overview

**Status**: Active Genesis Records  
**Last Updated**: 2026-07-02

## Lineage Purpose

Lineage documentation maintains complete transparency and accountability by recording:
- All contributors and their roles
- All hardware used in operations
- Temporal markers and epochs
- Formal ceremonies and events

## Core Lineage Documents

### 1. Contributor Registry (`lineage/contributor-registry.md`)
Permanent record of all project participants:
- Contributor names and roles
- Contribution types and areas
- Verification details (GitHub, PGP keys)
- Date joined
- Current status
- IP rights declarations

**Rights & Responsibilities**:
- Permanent attribution
- IP ownership of contributions
- Governance participation
- Charter compliance

### 2. Device Registry (`lineage/device-registry.md`)
Tracking of all hardware involved in cryptographic operations:
- Device identification
- Owner information
- Tier classification (Critical, Primary, Verification)
- Keys stored
- Ceremony usage
- Security requirements

**Three Tiers**:
- **Tier 1**: Critical (air-gapped, high security)
- **Tier 2**: Primary (contributor devices)
- **Tier 3**: Verification (read-only, community accessible)

### 3. Epoch Ledger (`lineage/epoch-ledger.md`)
Temporal markers and governance periods:
- Epoch definitions and dates
- Root hashes at epoch boundaries
- Blockchain anchors
- Ceremonies within epoch
- Amendments enacted
- Milestones achieved

**Current Epoch**: Genesis (2026-07-02 - 2026-12-31)

### 4. Ceremonies Log (`lineage/ceremonies.md`)
Formal governance and cryptographic events:
- Ceremony identification and dates
- Participants and witnesses
- Actions taken
- Documents signed
- Blockchain anchors
- Cryptographic verification

**Ceremony Types**:
- Governance ceremonies (charter, amendments)
- Technical ceremonies (signing, hashing)
- Milestone ceremonies (epoch transitions)
- Dispute ceremonies (appeals, sanctions)
